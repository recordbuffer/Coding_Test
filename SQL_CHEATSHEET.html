기본 골격 · 실행 순서

작성 순서 (외워라)
SELECT   컬럼
FROM     테이블
WHERE    행 조건       # 집계 전 필터
GROUP BY 묶을 컬럼
HAVING   그룹 조건     # 집계 후 필터
ORDER BY 정렬 LIMIT n
※ 실제 실행은 FROM→WHERE→GROUP BY→HAVING→SELECT→ORDER BY. 그래서 WHERE에선 별칭(AS) 못 씀, HAVING·ORDER BY에선 됨

SELECT · 정렬 · 상위 n

SELECT * FROM ANIMAL_INS;
SELECT ANIMAL_ID, NAME  # 특정 컬럼
SELECT DISTINCT ANIMAL_TYPE # 중복 제거
정렬 (기본 오름차순)
ORDER BY NAME              # 오름 ASC
ORDER BY DATETIME DESC    # 내림
ORDER BY NAME, DATETIME DESC # 1순위 이름, 같으면 시각 내림
상위 n개 (가장 먼저/큰 것)
ORDER BY DATETIME LIMIT 1
# "가장 먼저 들어온" = 정렬 후 LIMIT

조건 · WHERE

= != <>  > < >= <=
AND / OR / NOT
범위 · 목록 · 패턴
BETWEEN 10 AND 20   # 10~20 (양끝 포함)
IN ('Dog','Cat')      # 여럿 중 하나
NOT IN (...)
NAME LIKE 'A%'      # A로 시작
NAME LIKE '%el%'    # el 포함
NAME LIKE '_a%'     # 2번째가 a (_=한 글자)

NULL 처리 (자주 나옴)

NULL은 = 로 못 비교한다
WHERE NAME IS NULL
WHERE NAME IS NOT NULL
NULL을 다른 값으로 대체
IFNULL(NAME, 'No name')   # MySQL
COALESCE(NAME, 'No name') # 표준, 첫 non-NULL
※ != 'Aged' 는 NULL 행을 빠뜨린다. NULL도 포함하려면 OR col IS NULL 추가

집계 · GROUP BY · HAVING

집계 함수
COUNT(*)          # 행 수 (NULL 포함)
COUNT(NAME)       # NULL 제외 개수
COUNT(DISTINCT ANIMAL_TYPE) # 종류 수
SUM MAX MIN AVG
묶어서 세기
SELECT ANIMAL_TYPE, COUNT(*)
FROM ANIMAL_INS
GROUP BY ANIMAL_TYPE;
그룹 조건 = HAVING
GROUP BY NAME
HAVING COUNT(*) >= 2  # 2번 이상 등장한 이름
※ 집계 결과 조건은 WHERE 아니라 HAVING

JOIN (테이블 연결)

양쪽 다 있는 것만 = INNER
SELECT i.ANIMAL_ID, i.NAME
FROM ANIMAL_INS i
JOIN ANIMAL_OUTS o        # INNER 생략형
  ON i.ANIMAL_ID = o.ANIMAL_ID;
왼쪽 전부 + 짝 없으면 NULL = LEFT
FROM ANIMAL_INS i
LEFT JOIN ANIMAL_OUTS o
  ON i.ANIMAL_ID = o.ANIMAL_ID
WHERE o.ANIMAL_ID IS NULL
# 들어왔지만 안 나간 동물 = 짝 없음
※ 테이블에 별칭(i,o) 붙이면 짧고 안 헷갈림

문자열 함수

CONCAT(a, '-', b)     # 이어붙이기
SUBSTR(s, 1, 3)       # 1번째부터 3글자
LENGTH(s)            # 길이
UPPER(s) LOWER(s)
REPLACE(s, 'a', 'b')
LEFT(s, 2) RIGHT(s, 2)
※ SQL 문자열 위치는 1부터 (0 아님)

날짜 함수 (DATE)

포맷 바꾸기 (제일 많이 씀)
DATE_FORMAT(DATETIME, '%Y-%m-%d')
# 2026-07-21 형태로
# %Y년 %m월 %d일 %H시 %i분 %s초
부분 추출 · 계산
YEAR(d) MONTH(d) DAY(d) HOUR(d)
DATEDIFF(a, b)       # 일수 차
DATE_ADD(d, INTERVAL 1 DAY)
NOW() CURDATE()
※ 날짜 비교는 WHERE d >= '2026-01-01' 처럼 문자열로도 됨

서브쿼리

값 하나와 비교 (스칼라)
WHERE DATETIME = (
  SELECT MAX(DATETIME) FROM ANIMAL_INS)
# 가장 최근 = 최댓값과 같은 행
목록에 포함 (IN)
WHERE ANIMAL_ID IN (
  SELECT ANIMAL_ID FROM ANIMAL_OUTS)
FROM 안 서브쿼리엔 별칭 필수
FROM (SELECT ... ) AS t

조건 분기 · CASE

SELECT NAME,
  CASE
    WHEN price >= 100 THEN 'high'
    WHEN price >= 50  THEN 'mid'
    ELSE 'low'
  END AS grade
FROM ...
※ 등급/구간 나누기 문제의 정석

자주 틀리는 것 · 핵심 5

1) NULL은 =/!= 로 비교 못 함 → IS NULL
2) 집계 후 조건은 HAVING (WHERE 아님)
3) != 조건은 NULL 행을 빠뜨림
4) WHERE엔 SELECT 별칭 못 씀
5) 문자열·날짜 위치·자리수는 1부터
MySQL 기준. 오라클은 IFNULL→NVL, LIMIT→ROWNUM 로 다름
